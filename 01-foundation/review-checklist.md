# Phase 1 Review Checklist (상세)

## 1) 정확성
- [ ] 요구사항 A/B/C를 모두 구현했는가?
- [ ] 경계 케이스(빈 입력, 동점, 사이클, 중복 이벤트)를 다루는가?
- [ ] 결과 검증 테스트가 충분한가?

## 2) 자료구조/알고리즘
- [ ] Top-K에 Heap을 선택한 근거가 명확한가?
- [ ] DFS/BFS 선택 기준이 요구사항과 연결되는가?
- [ ] 시간/공간 복잡도 분석이 문서화되어 있는가?

## 3) Java 코드 품질
- [ ] 도메인 의미가 드러나는 네이밍인가?
- [ ] 불변 객체/부수효과 최소화 원칙 준수 여부
- [ ] Optional/Stream 남용 없이 가독성이 유지되는가?
- [ ] 예외를 삼키지 않고, 원인/문맥 로그가 남는가?

## 4) 테스트 품질
- [ ] 단위 테스트 최소 15개 이상인가? (실습 합산)
- [ ] 성공/실패/경계 케이스 균형이 맞는가?
- [ ] 재현 가능한 입력 데이터가 제공되는가?

## 5) 운영성
- [ ] 구조화 로그(traceId, eventId, orderId, sku) 포함 여부
- [ ] 재시도 정책/중단 정책이 문서화되어 있는가?
- [ ] 성능 지표 측정 방법과 해석이 합리적인가?
