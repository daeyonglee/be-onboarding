# Phase 1 Review Checklist — Foundation 파트별 점검표

## 0) 학습 범위 인식
- [ ] 자료구조/알고리즘/Java 파트별 강점/약점을 구분해 기록했는가?

## 1) 자료구조
- [ ] List/Map/Set/Deque/Heap 선택 근거가 있는가?
- [ ] 시간복잡도 + 메모리 사용량을 함께 설명했는가?
- [ ] 대안 2개 이상과 비교했는가?

## 2) 알고리즘
- [ ] Top-K의 Sort/Heap 결과가 일치하는가?
- [ ] DFS/BFS 사이클 검출이 동작하는가?
- [ ] Sliding Window를 naive 방식과 비교했는가?
- [ ] Binary Search 결정 문제를 종료조건까지 정의했는가?

## 3) Java Core
- [ ] `record`/`enum`/VO 사용이 타당한가?
- [ ] 비즈니스/시스템 예외가 분리되어 있는가?
- [ ] 로그 필드(traceId/eventId/sku/type)가 누락되지 않는가?
- [ ] 코드 가독성(네이밍/메서드 길이/상수화)이 충분한가?

## 4) 안정성
- [ ] 멱등성(eventId) 처리로 중복 이벤트가 무해한가?
- [ ] 재고 음수 방지가 보장되는가?
- [ ] 재시도 실패 시 DLQ 전송이 검증되는가?

## 5) 테스트
- [ ] 테스트 총 30개 이상인가?
- [ ] 정상/경계/실패 케이스가 균형적인가?
- [ ] 핵심 버그 재현 테스트가 있는가?

## 6) 성능/분석
- [ ] List vs Map 비교 수치가 있는가?
- [ ] Sort vs Heap 비교 수치가 있는가?
- [ ] Naive vs Sliding Window 비교 수치가 있는가?
- [ ] 측정 환경과 한계를 문서화했는가?

## 7) 학습 회고
- [ ] 배운 점 5개를 실제 코드 사례와 연결했는가?
- [ ] 헷갈리는 점 3개를 다음 액션으로 연결했는가?
- [ ] 4주 후속 학습 계획이 현실적인가?
