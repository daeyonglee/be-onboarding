# Phase 4 Review Checklist (상세)

## 1) 패턴 적용
- [ ] Strategy/Factory 분리가 OCP를 실제로 만족하는가?
- [ ] Template Method 공통 흐름과 훅이 적절히 분리되었는가?
- [ ] Adapter 경계가 외부 API 변경을 격리하는가?

## 2) Clean Architecture
- [ ] UseCase가 프레임워크 의존 없이 유지되는가?
- [ ] Controller/Adapter에 비즈니스 로직이 새지 않는가?
- [ ] 의존성 방향 규칙이 코드 구조에 반영되는가?

## 3) Clean Code
- [ ] 메서드 길이/중첩도가 합리적으로 개선되었는가?
- [ ] 네이밍이 도메인 의도를 정확히 드러내는가?
- [ ] Value Object 도입으로 primitive obsession이 줄었는가?

## 4) 테스트
- [ ] 전략별 단위 테스트가 충분한가?
- [ ] 상태 전이 실패 케이스가 테스트되는가?
- [ ] 리팩토링 전후 회귀 테스트가 통과하는가?

## 5) 지표/문서
- [ ] 복잡도/중복/커버리지 개선 수치가 제시되는가?
- [ ] 전이표/프로모션 규칙 문서가 코드와 일치하는가?
- [ ] 트레이드오프와 한계가 보고서에 기술되는가?
