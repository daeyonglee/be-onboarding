# Phase 4 Review Checklist — 품질/패턴 확장 점검표

## 0) 학습 범위 인식
- [ ] 현재 수준/다음 목표를 문서로 분리했는가?

## 1) 패턴 적용
- [ ] Strategy/Factory 분리가 OCP를 실제로 만족하는가?
- [ ] Template Method 공통 흐름/훅 분리가 적절한가?
- [ ] Adapter 경계가 외부 변경을 격리하는가?
- [ ] 패턴 도입이 과설계가 아닌가(복잡도 대비 타당한가)?

## 2) Clean Architecture
- [ ] UseCase/Domain이 프레임워크 비의존적인가?
- [ ] Controller/Adapter로 비즈니스 로직 누수가 없는가?
- [ ] 의존성 규칙이 자동검증(ArchUnit 등)되는가?

## 3) Clean Code
- [ ] 메서드 길이/중첩도가 개선됐는가?
- [ ] 네이밍이 도메인 의도를 정확히 드러내는가?
- [ ] Value Object로 primitive obsession이 줄었는가?
- [ ] 매직 넘버/중복 분기가 제거됐는가?

## 4) 테스트
- [ ] 테스트 총 45개 이상인가?
- [ ] 전략별/조합/충돌 케이스가 충분한가?
- [ ] 상태 전이 실패 케이스가 검증되는가?
- [ ] adapter 계약/회귀 테스트가 있는가?

## 5) 지표/문서
- [ ] 커버리지/복잡도/중복 개선 수치가 있는가?
- [ ] 전이표/프로모션 룰 문서가 코드와 일치하는가?
- [ ] 리팩토링 전후 근거(diff/metric)가 있는가?
- [ ] 트레이드오프와 한계를 명시했는가?

## 6) 회고
- [ ] 품질 부채 3개를 식별했는가?
- [ ] 다음 4주 개선 계획이 현실적인가?
- [ ] 팀 공유 가능한 품질 가이드를 작성했는가?
