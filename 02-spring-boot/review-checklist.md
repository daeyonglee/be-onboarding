# Phase 2 Review Checklist (상세)

## 1) 유스케이스 정확성
- [ ] 주문 생성 성공/실패 흐름이 요구사항과 일치하는가?
- [ ] 상태 전이(PENDING/PAID/FAILED)가 누락 없이 기록되는가?
- [ ] 예외 메시지가 사용자/운영자 관점에서 명확한가?

## 2) 설계/트랜잭션
- [ ] Controller-Service-Domain 경계가 분명한가?
- [ ] 트랜잭션 범위가 과도하지 않은가?
- [ ] 외부 결제 호출 실패 시 보상 전략이 있는가?
- [ ] ADR에 선택 근거와 대안 비교가 있는가?

## 3) JPA/성능
- [ ] N+1 개선 전/후 비교 근거(SQL 수)가 있는가?
- [ ] 페이징 전략 선택 이유가 문서화되었는가?
- [ ] 불필요한 eager loading이 없는가?

## 4) 테스트
- [ ] 도메인 정책 단위 테스트가 충분한가?
- [ ] 통합 테스트로 롤백/커밋 시나리오를 검증했는가?
- [ ] API 테스트에 실패 케이스(4xx/5xx)가 포함되는가?

## 5) 운영성
- [ ] traceId/orderId/userId가 모든 주요 로그에 있는가?
- [ ] 지표(실패율, p95, 재시도 횟수) 수집 경로가 있는가?
- [ ] 장애 대응(runbook/알람 기준)이 명시되었는가?
