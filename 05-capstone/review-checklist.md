# Phase 5 Review Checklist — Capstone 확장 점검표

## 0) 학습 범위 인식
- [ ] 현재 수준과 다음 성장 목표를 문서화했는가?

## 1) 기능 정확성
- [ ] 주문/결제/재고 유스케이스가 요구사항대로 동작하는가?
- [ ] 상태 전이 규칙이 코드/문서/테스트에서 일치하는가?
- [ ] 실패 시나리오에서도 정합성이 유지되는가?

## 2) 아키텍처
- [ ] Context/Aggregate/Port 경계가 명확한가?
- [ ] 외부 의존성이 Adapter로 격리되었는가?
- [ ] ADR이 실제 코드 구조와 트레이드오프를 설명하는가?

## 3) 테스트
- [ ] 테스트 총 60개 이상인가?
- [ ] 단위/통합/API/장애 주입 테스트가 균형적인가?
- [ ] 회귀를 방지할 핵심 시나리오가 포함되는가?

## 4) 성능/운영
- [ ] p95/p99/성공률 지표를 수집하는가?
- [ ] traceId/orderId/eventId가 일관되게 로그에 남는가?
- [ ] 재시도/DLQ/알람 기준이 문서화되었는가?
- [ ] MTTR 측정과 개선 계획이 있는가?

## 5) 문서/발표
- [ ] 문제-해결-효과 흐름이 명확한가?
- [ ] 전후 지표가 수치/그래프로 제시되는가?
- [ ] 한계와 후속 개선 항목이 현실적인가?

## 6) 회고
- [ ] 기술/운영 부채 3개 이상 식별했는가?
- [ ] 다음 4주 액션 플랜이 구체적인가?
- [ ] 팀 공유 가능한 재사용 템플릿이 남았는가?
