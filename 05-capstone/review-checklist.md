# Phase 5 Review Checklist — Capstone

## 1) 기능 정확성
- [ ] 주문/결제/재고 핵심 유스케이스가 요구사항대로 동작하는가?
- [ ] 상태 전이 규칙이 코드/문서/테스트에서 일치하는가?
- [ ] 실패 시나리오에서 정합성 깨짐이 없는가?

## 2) 아키텍처
- [ ] 경계(Context/Aggregate/Port)가 명확한가?
- [ ] 외부 의존성이 Adapter로 격리되었는가?
- [ ] ADR이 실제 코드 구조를 설명하는가?

## 3) 테스트
- [ ] 단위/통합/API 테스트 균형이 적절한가?
- [ ] 장애 주입 테스트가 실질적 리스크를 검증하는가?
- [ ] 테스트가 회귀를 방지할 수준으로 신뢰 가능한가?

## 4) 성능/운영
- [ ] p95/p99/성공률 지표를 수집하고 있는가?
- [ ] 로그에 traceId/orderId/eventId가 일관되게 있는가?
- [ ] 재시도/DLQ/알람 기준이 문서화되었는가?

## 5) 발표/문서
- [ ] 문제-해결-효과 흐름이 명확한가?
- [ ] 전/후 지표가 수치로 제시되는가?
- [ ] 한계와 후속 개선 항목이 정리되어 있는가?
